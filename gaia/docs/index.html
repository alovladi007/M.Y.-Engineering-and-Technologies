Getting Started

Welcome to GAIA

GAIA (Green Adaptive Infrastructure Automation) is a comprehensive Kubernetes-native energy optimization platform that reduces data center energy consumption by 15-30% while maintaining performance SLAs.

Key Features

  • • Dynamic power management with PID control
  • • Adaptive batching for latency optimization
  • • Energy-aware autoscaling with KEDA
  • • Green window carbon-aware scheduling
  • • Closed-loop TensorRT-LLM tuning
  • • Comprehensive monitoring and reporting

Prerequisites

  • • Kubernetes 1.24+
  • • Helm 3.0+
  • • NVIDIA GPU Operator (for GPU nodes)
  • • Prometheus and Grafana (included in deployment)

Installation

Quick Start

Deploy GAIA with a single command using our comprehensive deployment script.

# Clone the repository
git clone https://github.com/yourorg/energy-orchestrator
cd energy-orchestrator

# Deploy everything
./deploy-all.sh

Manual Installation

1. Build Images

make build REG=ghcr.io/yourorg/energy-orchestrator TAG=v0.7.0

2. Deploy Stack

helm upgrade --install energy ./infra/charts/energy \
  --namespace energy-system \
  --create-namespace

3. Apply Policies

kubectl apply -f policies/
kubectl apply -f examples/

Configuration

Environment Variables

Configure GAIA using environment variables in your values.yaml file.

# Core optimization settings
alerts:
  targetJoulesPerReq: 200  # J/request target

# Green windows (UTC)
greenWindow:
  windows: "00:00-06:00,22:00-24:00"
  powerRelaxPercent: 5

# TRT-LLM tuning
trtllmTuner:
  targetP95Seconds: 0.2
  batchMin: 1
  batchMax: 8

Energy Policies

Create custom energy optimization policies using the EnergyPolicy CRD.

apiVersion: energy.io/v1alpha1
kind: EnergyPolicy
metadata:
  name: inference-optimization
spec:
  targetNamespace: default
  powerCaps:
    gpu: 250
  sla:
    p95LatencyMs: 200
  autoscaling:
    targetJoulesPerRequest: 150

API Reference

Authentication

All API requests require authentication using Bearer tokens.

curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.gaia.energy/v1/metrics

Endpoints

GET /v1/metrics
Get energy metrics
POST /v1/policies
Create policy
PUT /v1/optimize
Trigger optimization

Monitoring

Access Dashboards

Access Grafana and Prometheus dashboards for monitoring energy optimization.

Grafana

kubectl -n energy-system \
  port-forward svc/energy-grafana 3000:80

Access at http://localhost:3000 (admin/prom-operator)

Prometheus

kubectl -n energy-system \
  port-forward svc/energy-prometheus 9090:9090

Access at http://localhost:9090

Key Metrics

  • • energy:cluster_joules_per_request - Cluster efficiency
  • • energy:app_joules_per_request - Per-app consumption
  • • node_gpu_power_watts - GPU power usage
  • • energy:app_p95_latency_seconds - Application latency

Troubleshooting

Common Issues

High Latency After Optimization

Reduce PID controller gains to be less aggressive.

kubectl -n energy-system edit configmap energy-controller-config
# Decrease kp, ki values

Insufficient Energy Savings

Enable more aggressive optimization settings.

helm upgrade energy ./infra/charts/energy \
  --set optimization.mode=aggressive \
  --set optimization.powerLimits.gpuFloor=100

Pods Not Scaling

Check KEDA metrics and configuration.

kubectl get scaledobject -A
kubectl describe scaledobject vllm-energy-scaler

Examples

Python SDK Example

import gaia

# Initialize client
client = gaia.Client(
    api_key="your-api-key",
    endpoint="https://api.gaia.energy"
)

# Get cluster metrics
metrics = client.get_metrics(
    cluster_id="prod-cluster-1",
    time_range="1h"
)

print(f"Energy efficiency: {metrics.joules_per_request} J/request")
print(f"Power consumption: {metrics.total_power}W")
print(f"GPU utilization: {metrics.gpu_utilization}%")

Generate Energy Report

# Generate energy savings report
kubectl -n energy-system create job \
  --from=job/energy-report-generator manual-report

# Wait for completion
kubectl -n energy-system wait \
  --for=condition=complete job/manual-report

# Extract PDF
POD=$(kubectl -n energy-system get pods \
  -l job-name=manual-report -o jsonpath='{.items[0].metadata.name}')
kubectl cp energy-system/$POD:/out/energy-report.pdf ./energy-report.pdf
